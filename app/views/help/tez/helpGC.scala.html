This analysis gauges your job's GC efficiency. It shows the ratio between the task GC time and task CPU time.

Example

Vertex Task GC

Severity: Critical
Number of tasks 2205
Avg task runtime (ms) 4832
Avg task CPU time (ms) 2321
Avg task GC time (ms) 55
Task GC/CPU ratio 0.023423

Suggestions

If your task spends a lot time on GC, your job generally needs some code optimization. Check the hive.auto.convert.join.noconditionaltask.size size to see if large hash tables are being used for map joins. If large number of columns are being inserted, then try reducing hive.exec.orc.default.buffer.size. If your job is inserting multiple partitions, then set the setting hive.optimize.sort.dynamic.partition to true
